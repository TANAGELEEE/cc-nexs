# 输出纪律（Output Discipline）

> 来源：Case-016 事故分析 + 日常观察（agent 在产出物中泄漏 chain-of-thought 和重复上下文导致文档臃肿、token 浪费）。

---

## 规则

### 1. 禁止内部推理泄漏

产出物（文档、评审结论、测试报告）中**禁止**出现：
- `<thinking>` / `<analysis>` / `<reasoning>` 标签内容
- Chain-of-thought 过程描述
- "我认为"、"让我分析"、"经过思考" 等第一人称推理叙述
- 上下文追溯（"在之前的对话中..."、"用户提到过..."）

产出物只包含：**结论、决策、状态、下一步行动**。

### 2. 单次评论/结论 2000 字符上限

任何单次**评审结论、issue 评论、状态报告**不超过 2000 字符。

**不适用于**：spec.md、repo-context.md、deploy.md、api-doc.md 等正式文档产出——这些按需求完整度决定长度，不受此限。

超出处理方式（适用于评论/结论类产出）：
- 详细内容写入独立文件（如 `sa-review-detail.md`）
- 评论中只放摘要 + 文件路径引用

### 3. 禁止重复回顾历史

每次产出**只包含增量信息**：
- 不重述前几轮已确认的事实
- 不复制粘贴之前的评审结论
- 引用历史结论时用文件路径引用，不内联全文

### 4. 结构化格式

产出物统一格式：
- 开头一句话 = 结论/状态
- 可选短列表 = 关键要点（≤ 5 项）
- 结尾 = 下一步行动或阻塞说明
